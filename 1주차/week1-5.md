###소프트웨어 개발 방법론이란? 
소프트웨어를 생산하는데 필요한 반복적인 과정들을 정리한 것. 
(구조적 프로그래밍, 객체지향, 고속 개발 방법론, 익스트림 프로그래밍(Agile), 스크럼)
		


> 애자일 개발 프로세스란?
			특정 개발 방법론을 가리키는 말은 아니고 애자일(기민한, 좋은것을 빠르고 낭비없게 만드는 것)개발을 가능하게 해 주는 다양한 방법론 전체를 일컫는 말.


> UML이란? : 통합 모델링 언어 / 표준화된 범용 모델링 언어 / 객체 지향 소프트웨어 집약 시스템을 개발할 때 산출물을 명세화, 시각화, 문서화할 때 사용

###TDD와 PDD
TDD(Test-driven development) : 테스트 주도 개발 / 매우 짧은 개발 사이클을 반복하는 소프트웨어 개발 프로세스.
> !!항상 테스트 하는 걸 습관화하자, 버그를 줄인다  스쿨 과정중에 테스트 하는 법 듣기.
	

PDD(Plan-driven development) : 계획 기반 개발 /  계획을 세우고 그 계획을 실천하는데에 많은 시간과 노력을 할애하는 개발 방법. (고전적인 방법)
	
	*PDD의 예시*
	- 형상관리 : SW개발 및 유지보수 과정에서 발생하는   소스코드, 문서, 인터페이스 등 각종 결과물에 대해 형상을 만들고, 이들 형상에 대한 변경을 체계적으로 관리, 제어하기 위한 활동

	- 버전관리 : 형상관리의 일부 / 일반적인 소프트웨어 소스 코드만의 형상을 관리하는 것(SVN, Git, Mercurial, CVS……)

#프로그래밍 언어(3가지)
###컴파일 언어(C, C++) : 컴파일 언어는 텍스트 형태의 소스 코드를 컴파일하여 기계어 형태로 된 실행 파일(바이너리)로 만든다. 

장점 : 실행 속도가 빠르다

단점 : 수정과 배포가 불편하다.

###바이트코드 언어(Java, C#) : 가상머신을 통해 기계어가 아닌 바이트코드 형태로 컴파일한다. 어느 CPU에서든지 같은 코드를 실행할 수 있다. 

장점: 범용적으로 모든 시스템에 적용 가능하다.(이식성이 높다)

단점: 가상머신을 무조건 설치 해야한다. 가상머신을 실행시켜야하므로 시스템의 리소스를 소모시킨다.

###인터프리터 언어(BASIC, JavaScript, Python, Ruby) : 텍스트 형태의 소스 코드를 한줄한줄 해석하면서 실행한다.

장점: 소스코드를 수정하기 쉽다.

단점 : 실행 속도가 느리다.

#프로그래밍 용어
####### 개발자란? 프로그래머와 기획, 디자이너 등 고객을 대상으로 하는 프로그램을 제작하는 데 필요한 직업군들을 총체적으로 가리키는 말이다.

####### Server/Client? 한쪽이 정보를 요청(client)하면 다른 한쪽이 정보를 제공(server)하는 관계. 

#######Front-end / back-end ? 사용자의 행동이 직접적으로 반영이 되는 인터페이스를 구성(Front-end) / 인터페이스 아닌 
#######Thread : 프로세스 내에서 실행되는 흐름의 단위
#######멀티 스레드 vs 멀티 태스크  vs 멀티 프로그래밍
	- 멀티 스레드 : 한 프로세스 내에서 여러 스레드가 동시에 생성, 실행된다. 또한 프로세스 내에 여러 스레드들은 공유 메모리를 통해 자원을 공유한다.
	- 멀티 태스크 : 태스크는 프로세스에서 확장된 개념으로서, 여러 개의 태스크를 동시에 실행하고 교대로 컴퓨터의 자원을 사용하도록 한다. 
	서로간의 자원이 공유되지 못하기 때문에 IPC(Inter-Process Communication)을 구현해야 한다.
	- 멀티 프로그래밍 ; 프로세서의 자원 낭비를 최소화하기 위해 낭비되는 시간을 다른 프로세스 수행에 쓰게 한다.
#######Library : 특정 기능을 수행할 수 있는 클래스 또는 함수의 집합체.	
#######API(Application Programming Interface) : 응용 소프트웨어와 프로그래밍 접합부 / 소프트웨어 간의 통신을 위해 메시지를 전달하는 방식 등이 결정된 것.
#######Framework : 기반과 비슷한 용어.
#######디자인 패턴 : 프로그램을 개발하는 과정에서 빈번하게 발생하는 디자인 상의 문제를 정리해서, 상황에 따라 간편하게 적용해서 쓸수 있는 패턴 형태로 만든 것.
> (대표적으로 MVC 패턴, Singleton 패턴이 있다. 공부할것!!!!!)
 
#######IDE(Integrated Development Environment)
#######SDK(Software Development Kit) : 소프트웨어 개발에 필요한 도구 모음(IDE+Framework+Tools………..)
